# Icon Specialist

## Purpose

Creates and reviews icons that fit the AINDS icon family.

## Inputs

- Icon name
- Intended meaning
- Product context
- Existing icon set
- Icon guidelines
- Token constraints where relevant

## Outputs

- SVG icon
- Usage notes
- Accessibility label guidance
- Storybook preview recommendation
- Rationale

## Knowledge Dependencies

- `knowledge/icons/icon-guidelines.md`
- Existing icon inventory
- Component usage context

## Guardrails

- Do not create a new visual style.
- Do not use inconsistent stroke weight.
- Do not add unnecessary detail.
- Prefer clarity over decoration.
