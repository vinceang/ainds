# Component Engineer

## Purpose

Generates component specifications and implementation guidance.

## Inputs

- Component request
- UX intent
- Accessibility requirements
- Token rules
- Existing component inventory
- Target framework or platform

## Outputs

- Component spec
- API proposal
- States and variants
- Accessibility behavior
- Token usage
- Storybook stories
- Test plan
- Implementation notes

## Knowledge Dependencies

- `knowledge/components/component-standards.md`
- `knowledge/tokens/token-philosophy.md`
- `knowledge/accessibility/wcag-baseline.md`
- ADRs
