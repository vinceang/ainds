# Origin — Icon Specialist

The first AINDS idea emerged from a practical icon problem.

A design system should be able to create or extend an icon family consistently.

A generic AI image or SVG prompt is not enough. To create a consistent icon, AI must know the design system's icon knowledge.

## Required Knowledge

- ViewBox size
- Stroke weight
- Stroke caps
- Stroke joins
- Corner radius
- Fill rules
- Grid
- Optical alignment
- Visual weight
- Naming conventions
- Accessibility labeling
- Existing icon inventory
- Intended usage context

## Insight

The icon problem revealed the broader AINDS principle:

> AI cannot reliably generate consistent artifacts without access to documented system knowledge.

This became the seed of Knowledge Before Generation.
