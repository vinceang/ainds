# Component Standards

Every component should include:

- Purpose
- Usage guidance
- Anatomy
- Variants
- States
- Accessibility requirements
- Keyboard behavior
- Token usage
- Responsive behavior
- i18n considerations
- Storybook stories
- Tests
- Review checklist

## Component Acceptance Criteria

A component is complete only when it is:

- Designed
- Implemented
- Tokenized
- Accessible
- Documented
- Tested
- Reviewed
