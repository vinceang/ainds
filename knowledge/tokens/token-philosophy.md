# Token Philosophy

AINDS is token-first.

Components should express design decisions through tokens rather than hard-coded values.

## Token Layers

```txt
Primitive Tokens
  ↓
Semantic Tokens
  ↓
Component Tokens
  ↓
Theme Tokens
  ↓
Product Implementations
```

## Principles

- Use semantic tokens in components.
- Avoid hard-coded colors, spacing, radii, shadows, and typography.
- Support light, dark, and high-contrast themes.
- Tokens should be readable by humans and machines.
- Tokens should be documented for AI consumption.
