# Contributing to AINDS

AINDS is built around one guiding principle:

> Knowledge Before Generation.

Contributions should strengthen the shared knowledge base before they expand generation capabilities.

## Contribution Guidelines

Before proposing a new agent, component, workflow, or tool, document:

1. What knowledge it depends on.
2. What source-of-truth files it should consult.
3. What human decision it supports.
4. How output will be reviewed.
5. How it reinforces Knowledge Before Generation.

## Definition of Done

A contribution is not complete until it includes:

- Documentation
- Rationale
- Accessibility considerations
- Governance impact
- AI-consumption guidance where relevant
- ADR if the decision changes architecture or methodology
