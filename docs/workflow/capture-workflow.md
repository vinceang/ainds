# Capture Workflow

AINDS must practice its own philosophy.

If the project is about transforming tribal knowledge into organizational knowledge, then project discoveries cannot remain trapped in chat.

## Rule

> Nothing important lives only in chat.

## Workflow

```txt
Conversation
    ↓
Session Note
    ↓
Classification
    ├── ADR
    ├── Book Chapter
    ├── Knowledge Article
    ├── Glossary Entry
    ├── Agent Definition
    └── Roadmap Item
    ↓
Review
    ↓
Permanent Artifact
```

## Session Close Checklist

At the end of each meaningful session, capture:

- What did we discover?
- What decisions did we make?
- What terms became important?
- What belongs in the book?
- What belongs in the knowledge base?
- What needs an ADR?
- What remains unresolved?
- What should we do next?

## Promotion Rules

Promote a note into a permanent artifact when:

- It changes direction.
- It defines a principle.
- It explains a recurring decision.
- It adds reusable knowledge.
- It affects humans and AI.
- It reinforces Knowledge Before Generation.
