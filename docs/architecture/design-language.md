# Design Language

## Core Idea

A design system should teach AI not only *what* to generate, but *how* to generate it, grounded in the organizational knowledge of *why*.

## Beyond Components

Traditional design systems define reusable assets.

AINDS defines reusable design language.

This includes:

- Visual language
- Interaction language
- Content language
- Motion language
- Icon language
- Illustration language
- Photography language
- Brand language

## Examples

### Icons

"Generate a new icon that looks like it has always belonged in the Lucide family."

### Illustrations

"Generate the next illustration in our existing illustration style."

### Copy

"Write this empty state using our product voice, sentence casing, terminology, and reading level."

### Components

"Create a new component that follows our spacing, accessibility, validation, and interaction philosophy."

## The AINDS Difference

Most AI systems generate assets.

AINDS extends design languages.

Every generated artifact should feel like the next logical member of the family—not an exception.

## Positioning

> A design system informs AI not just *what* to build, but *how* to build it with the knowledge of *why*.
